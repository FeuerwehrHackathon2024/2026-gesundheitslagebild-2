Problem & Idee

Im MANV-Fall (Massenanfall von Verletzten) kommen ~250 Verletzte/Tag an einem zentralen Hub an. Die Leitstelle muss sie triagieren, auf passende Krankenhäuser verteilen, Transporte organisieren und das Lagebild in Echtzeit führen.

  • 72 h Vorwarnung — planvolle Verteilung möglich
  • Triage nach SK1/SK2/SK3 (vital · schwer stabil · leicht)
  • Aufenthaltsdauer: SK1 = 5 Tage · SK2 = 3 Tage · SK3 = 1 Tag
  • Ziel: Software, die Leitstelle + Rettungsdienst + Krankenhäuser koordiniert

HL7-Integration (i-engineers-Connector)

Der Connector der Kollegen von i-engineers pushed HL7 v2.5 ADT-Nachrichten direkt in unsere /api/adt/ingest Endpoint. Wir verstehen:

  • A01 Aufnahme → belegung_sk[n] += 1
  • A03 Entlassung → belegung_sk[n] -= 1
  • A08 Update → Log-Only

SK wird aus PV1-19 (Visit Number, unsere Konvention) oder PV1-10 (Hospital Service, Kollegen-Konvention) gelesen. Fachabteilungs-Codes (ITS, UNF, ORT, KAR, …) werden über die FA_TO_SK-Tabelle zu MANV-SK übersetzt.

CLI-Beispiele
# 50 Aufnahmen an VM senden python3 scripts/hl7_adt_generator.py --event A01 -n 50 \ --post http://185.181.169.154/api/adt/ingest # Kontinuierliche Last (1 msg/s, Mix aus A01/A08/A03) python3 scripts/hl7_adt_generator.py --mix -n 500 --rate 1 \ --post http://185.181.169.154/api/adt/ingest

SK ↔ IVENA ↔ Transportmittel

SKIVENA (röm.)FarbeBedeutung AufenthaltTransportmittelKap. pro Fahrt
SK1 I rot vital gefährdet / lebensbedrohlich 5 Tage RTW — Rettungswagen + Notarzt 1 Pat
SK2 II gelb schwer verletzt, stabil / dringlich 3 Tage KTW — Krankentransportwagen 1 Pat
SK3 III grün leicht verletzt / aufgeschoben 1 Tag Taxi oder BTW 2 Pat (gebündelt)

SK-Kompatibilität: Eine SK1-fähige Klinik kann auch SK2/SK3 versorgen. Eine SK2-Klinik kann SK2+SK3. SK3-Klinik nur SK3.

Datenpipeline

1. Amtlicher Qualitätsbericht 20242.273 Zeilen mit IK, Fachabteilungen, Notfallstufe
2. QraGo-DB3.728 Google-Places + LLM-Enrichment (Kapazität, Score)
3. Merge scripts/merge_krankenhaeuser.pyper (IK, Standortnummer) — 4.420 Zeilen
4. SK-Ableitung scripts/add_sk_capability.pykann_sk1/2/3 aus Fachabteilungen + Intensivmed. + Notaufnahme + Uniklinik
5. Deduplizierung scripts/deduplicate_krankenhaeuser.py4.420 → 2.081 eindeutige Akut-Kliniken (Name-Norm + Geo <500m + MANV-Filter)
6. Auto-Seed beim App-Start hackathon/seed.pyKrankenhaus-Tabelle + Hub "Hub Süd" (Ulm) werden initial befüllt

Deployment

VM: 185.181.169.154 · Ubuntu 24.04 · 4 CPU / 8 GB RAM · Docker + Docker-Compose

# Erstinstallation ssh ubuntu@185.181.169.154 cd ~/hackathon_krankenhaus cp .env.example .env nano .env # Passwörter + HERE_API_KEY eintragen sudo docker compose up -d --build # Live-Logs sudo docker compose logs -f web # Nur web neu bauen sudo docker compose build web && sudo docker compose up -d
.env-Variablen
  • SECRET_KEY — Flask-Sessions
  • POSTGRES_USER / POSTGRES_PASSWORD / POSTGRES_DB
  • HERE_API_KEY — aktiviert Routing + Traffic (Fallback: Haversine)
  • WEB_PORT — Host-Port (default 80)
